# CMOS 自动截图生成 PPT 工程

这是一个面向 CMOS 课程实验/仿真整理的小型自动化工具。项目的目标是减少手工重复操作：从已经打开的课件、仿真页面或实验结果窗口中自动截取指定区域，按页保存为连续图片，便于后续整理到 PPT 中。

当前仓库的主要脚本聚焦在“自动截图采集”这一步；PPT 成品、课程讲义 PDF 和个人运行输出不随仓库上传。

## 功能特点

- 支持交互式输入章节、标题、页数和翻页间隔。
- 支持命令行参数运行，便于复现实验截图流程。
- 支持手动框选截图区域，并在本地保存区域配置。
- 自动按章节创建输出目录，截图文件按 `001.png`、`002.png` 顺序命名。
- 每截一页后自动发送键盘右方向键，适合可用方向键翻页的课件或页面。
- 不连接浏览器、不登录网站、不解析网页内容，避免把账号流程写入代码。

## 使用环境

- 操作系统：Windows
- Python：建议 Python 3.10 或更新版本
- 运行前需要手动打开目标课件、实验页面或仿真结果窗口
- 目标窗口需要支持键盘右方向键翻页


## 运行方法

交互式运行：

```powershell
python capture_slides.py
```

程序会依次询问：

```text
请输入章节编号：
请输入章节名称：
请输入PPT总页数：
请输入翻页等待时间（秒）：
是否框选区域？(y/n)：
```

也可以直接使用命令行参数：

```powershell
python capture_slides.py --chapter 2 --title "Basic MOS Device Physics" --pages 50 --interval 0.5 --select-region
```

参数说明：

- `--chapter`：章节编号，例如 `2`
- `--title`：章节或实验名称，例如 `"Basic MOS Device Physics"`
- `--pages`：需要截图的总页数
- `--interval`：翻页前后的等待时间，单位为秒
- `--select-region`：重新框选截图区域，并覆盖本地保存的区域配置

## 使用流程

1. 手动打开目标课件、实验页面或仿真结果窗口。
2. 将目标内容停留在第一页。
3. 运行 `capture_slides.py`。
4. 第一次运行或需要重新定位时，选择框选区域。
5. 按提示点击截图区域左上角和右下角。
6. 在倒计时期间切回目标窗口。
7. 程序开始截图并自动翻页。

截图过程中不要移动目标窗口，也不要改变浏览器缩放、页面布局或显示器分辨率。

## 输入/输出说明

输入来源：

- 已经打开并停留在第一页的课件、实验页面或仿真结果窗口
- 用户输入的章节编号、标题、页数和翻页间隔
- 本地截图区域配置 `capture_region.json`

输出目录：

```text
captures/第2章Basic MOS Device Physics/
```

输出文件示例：

```text
001.png
002.png
003.png
...
050.png
```

`captures/` 是运行输出目录，默认不会提交到 GitHub。

## 本地配置

`capture_region.json` 会记录本机屏幕坐标，例如：

```json
{
  "x1": 100,
  "y1": 100,
  "x2": 1200,
  "y2": 800
}
```

这个文件与个人显示器、窗口位置有关，默认不提交。仓库中保留 `capture_region.example.json` 作为格式参考。

## 常见问题

### 截图范围不准确

重新运行并增加 `--select-region`：

```powershell
python capture_slides.py --select-region
```

### 截图重复

通常是翻页后等待时间太短。可以把 `--interval` 从 `0.2` 调整为 `0.5` 或更高。

### 翻页失败

确认目标窗口已经在倒计时结束前切到前台，并且该页面支持键盘右方向键翻页。

## 设计边界

这个项目是课程自动化整理工具，不做网页爬虫、账号登录、OCR、AI 图像识别或课件内容解析。核心思路是用稳定、可复现的本地操作，把重复截图流程自动化。
