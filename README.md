# AutoSlideCapture

AutoSlideCapture 是一个 Windows 本地自动截图工具，用于把已经打开的演示资料、网页幻灯片、实验记录或仿真结果窗口按页截取为连续图片。它适合处理需要重复“截图、翻页、等待、保存”的场景，后续可以把生成的图片整理到 PPT、报告或归档目录中。

当前仓库的主要脚本聚焦在“自动截图采集”这一步；截图输出、PPT 成品、PDF 资料和个人运行配置不随仓库上传。

## 功能特点

- 支持交互式输入编号、标题、页数和翻页间隔。
- 支持命令行参数运行，便于复现固定截图流程。
- 支持手动框选截图区域，并在本地保存区域配置。
- 自动创建输出目录，截图文件按 `001.png`、`002.png` 顺序命名。
- 每截一页后自动发送键盘右方向键，适合可用方向键翻页的窗口或页面。
- 不连接浏览器、不登录网站、不解析网页内容，避免把账号流程写入代码。

## 使用环境

- 操作系统：Windows
- Python：建议 Python 3.10 或更新版本
- 运行前需要手动打开目标窗口
- 目标窗口需要支持键盘右方向键翻页

## 安装依赖

进入项目目录：

```powershell
cd path\to\AutoSlideCapture
```

可选：创建并启用虚拟环境。

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

安装依赖：

```powershell
pip install -r requirements.txt
```

## 运行方法

交互式运行：

```powershell
python capture_slides.py
```

程序会依次询问：

```text
请输入编号：
请输入标题：
请输入总页数：
请输入翻页等待时间（秒）：
是否框选区域？(y/n)：
```

也可以直接使用命令行参数：

```powershell
python capture_slides.py --chapter 1 --title "Demo Slides" --pages 20 --interval 0.5 --select-region
```

参数说明：

- `--chapter`：编号，例如 `1`
- `--title`：标题或任务名称，例如 `"Demo Slides"`
- `--pages`：需要截图的总页数
- `--interval`：翻页前后的等待时间，单位为秒
- `--select-region`：重新框选截图区域，并覆盖本地保存的区域配置

## 使用流程

1. 手动打开目标窗口。
2. 将目标内容停留在第一页。
3. 运行 `capture_slides.py`。
4. 第一次运行或需要重新定位时，选择框选区域。
5. 按提示点击截图区域左上角和右下角。
6. 在倒计时期间切回目标窗口。
7. 程序开始截图并自动翻页。

截图过程中不要移动目标窗口，也不要改变浏览器缩放、页面布局或显示器分辨率。

## 输入/输出说明

输入来源：

- 已经打开并停留在第一页的目标窗口
- 用户输入的编号、标题、页数和翻页间隔
- 本地截图区域配置 `capture_region.json`

输出目录示例：

```text
captures/1_Demo Slides/
```

输出文件示例：

```text
001.png
002.png
003.png
...
020.png
```

`captures/` 是运行输出目录，默认不会提交到 GitHub。

## 本地配置

`capture_region.json` 会记录本机屏幕坐标，例如：

```json
{
  "x1": 100,
  "y1": 100,
  "x2": 1200,
  "y2": 800
}
```

这个文件与个人显示器、窗口位置有关，默认不提交。仓库中保留 `capture_region.example.json` 作为格式参考。

## 常见问题

### 截图范围不准确

重新运行并增加 `--select-region`：

```powershell
python capture_slides.py --select-region
```

### 截图重复

通常是翻页后等待时间太短。可以把 `--interval` 从 `0.2` 调整为 `0.5` 或更高。

### 翻页失败

确认目标窗口已经在倒计时结束前切到前台，并且该页面支持键盘右方向键翻页。

## 设计边界

AutoSlideCapture 是一个轻量的本地截图自动化工具，不做网页爬虫、账号登录、OCR、AI 图像识别或页面内容解析。核心思路是用稳定、可复现的本地操作，把重复截图流程自动化。
